## Sampling Architecture

**Key design decision:** The adapter uses ALL samples from ImportanceSampler (training + local + global regions), NOT just training points.

Why this matters:
- `induced_kl_divergence` evaluates the candidate expression across the entire sampled distribution
- This includes training data, local exploration (KDE), and global exploration (uniform) regions
- The fitness function computes GECCO weights internally based on reference PDF values
- We do NOT pass `sampler.weights` to PhySO via `y_weights` because:
  1. GECCO weights already capture importance via reference PDF
  2. Double-weighting (sampler.weights + GECCO weights) would incorrectly bias fitness
  3. This matches Julia reference implementation (fitness.py lines 96-106)

**Data flow:**
```
PhySO training → evaluates expression → produces y_pred (len = sampler.samples)
                                    ↓
              adapter.kl_reward(y_target, y_pred, y_weights=1.)
                                    ↓
                   Validates: len(y_pred) == len(sampler.samples)
                                    ↓
                   induced_kl_divergence(f_vals=y_pred, sampler=sampler, ...)
                                    ↓
                   Computes GECCO weights from reference_dist.pdf(sampler.samples)
                                    ↓
                   Returns exp(kl_divergence) → transformed to reward [0,1]
```
